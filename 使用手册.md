# RobotProject 使用手册

本文档面向当前仓库 `RobotProject` 的实际代码，说明系统思路、模块边界、常用命令与下一步方向。

外部来源与版权边界见 `外部来源说明.md`。

---

## 1. 项目整体思路（一句话）

把机器人任务拆成可独立迭代的四段：  
**感知（看见并编号）→ 认知（理解并决策）→ 运动适配（语言变向量）→ 扩散策略执行（连续动作）**，  
并在失败时进入反思闭环，把新意图重新注入策略。

---

## 2. 系统总览（3+1）

| 层级 | 核心文件 | 输入 | 输出 | 职责边界 |
|------|------|------|------|------|
| System 1 感知 | `perception_engine.py` | RGB、Depth、waypoint | `scene_json`（objects/id/class/3d） | 只做检测、分割、3D 锚定，不做语言推理 |
| System 2 认知 | `cognitive_brain.py` | 用户指令 + `scene_json` | `{"action","target_id"}`（可含 `macro_action`） | 只做任务理解、目标选择、失败反思 |
| System 3 运动适配 | `motor_adapter.py` | 决策 JSON | CLIP 文本嵌入 `(1,512)` | 把离散语义变成可注入策略的向量 |
| +1 小脑执行 | `ADM_DP/policy/Diffusion-Policy/*` | 图像/状态/语言向量 | 关节动作序列 | 连续控制与时序规划 |

常用闭环入口：
- `test_pipeline.py`：极简冒烟链路。
- `vla_closed_loop_eval.py`：3+1 闭环评测（含失败反思 + 二次尝试）。
- `ADM_DP/policy/Diffusion-Policy/eval_real_sim.py`：纯 DP 仿真执行。

---

## 3. 关键数据流

```text
ManiSkill obs(rgbd/state)
  -> perception_engine.ManiSkillPerceptionWrapper.perceive()
  -> reference_library
  -> scene_json(objects with id/class/centroid)

user instruction + scene_json
  -> cognitive_brain.CognitiveBrain.decide()
  -> {action, target_id}

decision
  -> motor_adapter.build_semantic_phrase_for_film()
  -> motor_adapter.encode_text_emb_for_film()
  -> language_emb (1,512)

obs + language_emb
  -> Diffusion Policy
  -> action

if failed
  -> reflect_on_failure / local VLM multimodal reflect
  -> new decision -> new language_emb -> retry
```

## 5. 运行前检查清单

- 使用 `robot_vla` 环境并在仓库根执行命令。
- SAM2 权重存在：`graduate_pro/src/vision_ai/vision_ai/models/sam2/sam2_hiera_large.pt`。
- YOLO 权重存在：`graduate_pro/src/vision_ai/vision_ai/models/yolo/yolov8n.pt`。
- 训练和 vLLM 不要同时占满同一张 GPU（避免 OOM）。

## 6. 高频命令

```bash
cd ~/RobotProject
source robot_vla/bin/activate
```

```bash
# 1) 下载视觉权重
python scripts/download_vision_weights.py
```

```bash
# 2) H5 -> Zarr（含 language_emb）
python scripts/convert_h5_to_dp_zarr.py \
  --h5_path ~/.maniskill/demos/PickCube-v1/teleop/trajectory.h5 \
  --out_zarr ADM_DP/data/zarr_data/PickCube_v1_lang_bridge.zarr \
  --instruction "Pick up the green cube" \
  --clip_device cuda
```

```bash
# 3) 训练 DP（先停 vLLM）
cd ADM_DP/policy/Diffusion-Policy
python train.py --config-name=robot_dp task=pickcube_lang_minimal \
  training.device=cuda:0 exp_name=PickCube_v1_lang_bridge \
  +task.dataset.batch_size=32 dataloader.batch_size=32 val_dataloader.batch_size=32
```

```bash
# 4) checkpoint 冒烟
python eval_ckpt_smoke.py \
  --ckpt_path checkpoints/PickCube_v1_lang_bridge/150.ckpt \
  --device cuda:0 \
  --instruction "Pick up the green cube"
```

```bash
# 5) 3+1 闭环评测（仓库根执行）
cd ~/RobotProject
python vla_closed_loop_eval.py \
  --ckpt_path ADM_DP/policy/Diffusion-Policy/checkpoints/PickCube_v1_lang_bridge/150.ckpt \
  --env_id PickCube-v1 \
  --device cuda:0 \
  --phase_steps 100 \
  --render_mode rgb_array \
  --vlm_base_url http://localhost:11435 \
  --vlm_model Qwen2-VL-7B-AWQ \
  --instruction "Pick up the green cube"
```

## 7. 当前进展与瓶颈

- 已打通 `language_emb` 在数据集、训练、推理中的链路。
- 已实现失败反思并可通过本地 vLLM 做多模态 recovery 建议。
- 已补齐 SAM2 权重，感知初始化不再因缺 checkpoint 失败。
- 当前主要瓶颈是闭环成功率：依赖 `scene_json` 质量、反思动作词表一致性、以及重试预算。

---

## 8. 下一步发展方向（建议优先级）

1. **先提升感知可用率（最高优先）**  
   目标是稳定得到 `scene_json.count > 0`，并保证 `target_id` 与真实对象 ID 对齐。

2. **统一反思动作词表到训练动作空间**  
   把 VLM 输出动作标准化到 `grasp/place/...`，避免 `tryAgain` 这类 out-of-vocab 指令。

3. **优化闭环节奏与重试预算**  
   调整 `phase_steps`、必要时在重试前 reset，提高“反思后还有足够步数”的概率。

4. **做数据与域随机增强**  
   增加多场景/多视角样本，降低仿真与执行分布偏差。

5. **补评测指标看板**  
   统计 `scene_count`、ID 命中率、反思触发率、二次成功率，形成可比较的迭代指标。

---

## 9. 常见问题

- `scene_json` 为空：优先检查输入画面是否有可检测目标、YOLO 阈值、相机分辨率与视角。
- vLLM/训练 OOM：避免并行抢同一 GPU。
- 反思后仍失败：先核对 `target_id` 是否存在于本轮 `scene_json.objects`。
