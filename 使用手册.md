# RobotProject 使用手册

本文档与当前仓库代码对齐，说明系统思路、目录、常用命令、数据与评测注意事项，以及 **Git/GitHub** 推送经验。

外部来源与版权边界见 `外部来源说明.md`。

---

## 1. 项目整体思路（一句话）

把任务拆成：**感知（看见并编号）→ 认知（理解并决策）→ 运动适配（语言变向量）→ 扩散策略执行（连续动作）**；失败时进入反思闭环，更新意图后再试。

---

## 2. 系统总览（3+1）

| 层级 | 核心文件 | 输入 | 输出 | 职责边界 |
|------|----------|------|------|----------|
| System 1 感知 | `perception_engine.py` | RGB、Depth、waypoint | `scene_json` | 检测、分割、3D 锚定；不做语言推理 |
| System 2 认知 | `cognitive_brain.py` | 指令 + `scene_json` | `action` / `target_id` 等 | 理解、选目标、失败反思 |
| System 3 运动适配 | `motor_adapter.py` | 决策语义 | `language_emb`（如 512 维） | 注入策略的条件向量 |
| +1 执行 | `ADM_DP/policy/Diffusion-Policy/*` | 图像、状态、语言 | 动作序列 | 扩散策略 + ManiSkill `step` |

常用入口：

| 脚本 | 说明 |
|------|------|
| `test_pipeline.py` | 极简冒烟（感知→大脑→motor） |
| `vla_closed_loop_eval.py` | 3+1 闭环 + 反思 + 可选 vLLM 多模态反思 + 视频/JSONL 日志 |
| `ADM_DP/policy/Diffusion-Policy/eval_real_sim.py` | 仅扩散策略 + ManiSkill（无 VLM） |
| `ADM_DP/policy/Diffusion-Policy/eval_ckpt_smoke.py` | 用 Zarr 前几步做 forward，对比 GT action（不开仿真） |

---

## 3. 关键数据流（闭环）

```text
ManiSkill obs(rgbd + robot state)
  ->（反思阶段）ManiSkillPerceptionWrapper.perceive()  [可选 sim_gt / hybrid]
  -> scene_json

instruction + scene_json -> CognitiveBrain / vLLM -> 决策
  -> build_semantic_phrase_for_film + encode_text_emb_for_film -> language_emb

obs（与训练一致的图像与状态）+ language_emb -> DP -> action -> env.step()

失败 -> reflect -> 新 phrase/emb ->（可选 reset）-> 再跑 Phase D
```

## 4. 目录与作用

| 路径 | 作用 |
|------|------|
| `graduate_pro/` | 视觉栈（YOLO/SAM2/3D 后处理），由 `perception_engine.py` 引用 |
| `ADM_DP/` | Diffusion Policy 训练、配置、评测脚本 |
| `scripts/convert_h5_to_dp_zarr.py` | ManiSkill H5 → Zarr；优先 **31 维 panda 状态** 与 RGB；可写 `language_emb` |
| `scripts/download_vision_weights.py` | 下载 YOLOv8n + SAM2 权重到 `vision_ai/models/` |
| `vla_closed_loop_eval.py` | 仓库根目录；VLA 闭环评测主入口 |
| `embodied_command_hub.py` | 语音入口（ASR + 指令解析） |

可选环境变量：`VISION_CONFIG` → 自定义 `enhanced_detection_config.json` 绝对路径（见 `perception_engine.py`）。

## 5. 数据与状态（训练/仿真必须一致）

1. **原始 teleop H5 常为 `obs_mode: none`**：`obs` 里可能没有 RGB。需要先用 ManiSkill **replay** 生成带 RGB 与 `env_states` 的 H5，再跑 `convert_h5_to_dp_zarr.py`。  
   - 示例（具体路径以你机器为准）：  
     `python -m mani_skill.trajectory.replay_trajectory --save-traj --obs-mode rgb --use-env-states -c pd_joint_pos ...`
2. **Zarr 中 `state`**：转换脚本优先使用 **`env_states/articulations/panda` 的 31 维**，与 ManiSkill `robot.get_state()` 一致。  
3. **仿真评测 `eval_real_sim.py` / `vla_closed_loop_eval.py`**：策略输入里的 **`agent_pos` 使用 `env.unwrapped.agent.robot.get_state()`**，与 Zarr 对齐；不要用手拼 `qpos+qvel+tcp+...` 代替。  
4. **图像**：PickCube 管线常用 **240×320** 与 Zarr `head_camera` (N,3,H,W) 一致；脚本侧会对齐或缩放。

## 6. 运行前检查

- 激活你装好 ManiSkill / torch 的环境（如 `conda activate robot_vla`；若只用 miniconda 的 base 也可，以你实际为准）。
- 感知链路：YOLO / SAM2 权重路径见 `scripts/download_vision_weights.py` 与 `graduate_pro/.../enhanced_detection_config.json`。
- **训练 DP 与 vLLM 尽量不要同卡满占**，避免 OOM。
- **vLLM**：`--vlm_model` 必须与服务的 **`--served-model-name`** 一致；可用 `GET {base}/v1/models` 核对 id。

## 7. 高频命令

以下路径以仓库根 `~/RobotProject` 为例；请按本机修改。

### 7.1 下载视觉权重

```bash
cd ~/RobotProject
python scripts/download_vision_weights.py
```

### 7.2 H5 → Zarr（含 `language_emb`）

将 `--h5_path` 换成你 **replay 后** 的 H5；`--out_zarr` 可指向 `ADM_DP/data/zarr_data/...`。

```bash
python scripts/convert_h5_to_dp_zarr.py \
  --h5_path ~/.maniskill/demos/PickCube-v1/teleop/trajectory.rgb.pd_joint_pos.physx_cpu.h5 \
  --out_zarr ADM_DP/data/zarr_data/PickCube_v1_lang_bridge.zarr \
  --instruction "Pick up the green cube" \
  --clip_device cuda
```

### 7.3 训练 Diffusion Policy

在 **`ADM_DP/policy/Diffusion-Policy`** 目录执行（训练前可先停 vLLM）：

```bash
cd ~/RobotProject/ADM_DP/policy/Diffusion-Policy
python train.py --config-name=robot_dp task=pickcube_lang_minimal \
  training.device=cuda:0 exp_name=PickCube_v1_lang_bridge \
  +task.dataset.batch_size=32 dataloader.batch_size=32 val_dataloader.batch_size=32
```

- **Hydra 日志与 config**：`ADM_DP/policy/Diffusion-Policy/data/outputs/<日期>/<时间>_robot_task_config_task_config/`  
- **周期性 checkpoint**：默认写在当前包目录下  
  `ADM_DP/policy/Diffusion-Policy/checkpoints/<exp_name>/`（如 `50.ckpt`、`100.ckpt`），**不在**上面的 `data/outputs/...` 里。

### 7.4 Checkpoint 冒烟（Zarr）

仍在 **`Diffusion-Policy`** 目录：

```bash
cd ~/RobotProject/ADM_DP/policy/Diffusion-Policy
python eval_ckpt_smoke.py \
  --ckpt_path checkpoints/PickCube_v1_lang_bridge/150.ckpt \
  --zarr_path ../../data/zarr_data/PickCube_v1_lang_bridge.zarr \
  --device cuda:0 \
  --instruction "Pick up the green cube"
```

也可用绝对路径指向 `.ckpt` 与 `.zarr`。

### 7.5 仅 DP 的 ManiSkill 闭环

```bash
cd ~/RobotProject/ADM_DP/policy/Diffusion-Policy
python eval_real_sim.py \
  --ckpt_path checkpoints/PickCube_v1_lang_bridge/150.ckpt \
  --max_steps 150 \
  --instruction "Pick up the green cube" \
  --render_mode rgb_array \
  --save_video eval_video/pickcube_eval.mp4
```

### 7.6 完整 VLA 闭环（需先启动 vLLM）

在 **仓库根目录**：

```bash
cd ~/RobotProject
python vla_closed_loop_eval.py \
  --ckpt_path ADM_DP/policy/Diffusion-Policy/checkpoints/PickCube_v1_lang_bridge/150.ckpt \
  --env_id PickCube-v1 \
  --device cuda:0 \
  --phase_steps 100 \
  --render_mode rgb_array \
  --vlm_base_url http://localhost:11435 \
  --vlm_model Qwen2-VL-7B-Instruct-AWQ \
  --instruction "Pick up the green cube" \
  --scene_source perception
```

**`--scene_source` 说明：**

| 取值 | 含义 |
|------|------|
| `perception` | 反思阶段走 YOLO+SAM2，生成 `scene_json`（默认） |
| `sim_gt` | 不加载检测模型；PickCube 用仿真 cube/goal 真值填 `scene_json`（调试用） |
| `hybrid` | 先感知；若空或失败则回退 `sim_gt` |

可选：`--perception_visualization` 打开检测管线可视化输出。

默认视频与反思日志路径见 `vla_closed_loop_eval.py` 内 `--video_path` / `--reflect_log_path`（支持相对仓库根的路径）。

## 8. Git / GitHub（无 sudo 场景）

- **没有 `sudo`** 时不要用 `apt install gh`，可用：**`conda install -c conda-forge gh -y`**，然后 `gh auth login`、`gh auth setup-git`。
- 若 `gh` 用 **HTTPS** 登录，但 `git remote -v` 仍是 **`git@github.com:...`**，`git push` 会报 `Permission denied (publickey)`。请改为：  
  `git remote set-url origin https://github.com/Samuel-0920/RobotProject.git`
- 推送若提示 **先 fetch**：`git fetch origin` → `git pull --rebase origin main` → `git push origin main`。
- **切勿**在聊天/截图中泄露 Personal Access Token；泄露后立刻在 GitHub 上 **Revoke** 并换新。

## 9. 当前进展与主要瓶颈

- 已打通 `language_emb` 在数据集、训练、推理与闭环中的链路。
- 已支持本地 vLLM 参与反思；`scene_source` 支持 `perception` / `sim_gt` / `hybrid`。
- 仿真侧状态向量已与 Zarr（`robot.get_state()`）对齐，避免训练/评测分布不一致。
- 瓶颈仍常见于：**闭环成功率**、`scene_json` 质量、反思动作与策略动作空间一致、以及步数/reset 预算。

---

## 10. 下一步建议（优先级）

1. 提高感知可用率（稳定 `scene_json.count > 0` 与 `target_id` 对齐）。  
2. 统一反思输出的动作词表与训练/策略可执行集合。  
3. 调节 `phase_steps`、重试前 `reset` 等闭环节奏。  
4. 数据与域随机化，减轻 sim–real 偏移。  
5. 建立成功率、反思触发率等评测指标，便于对比迭代。

## 11. 常见问题

| 现象 | 排查方向 |
|------|----------|
| Zarr 全黑图或没有 RGB | 源 H5 是否未 replay；是否选了带 `sensor_data/.../rgb` 的轨迹 |
| `state` 维数不是 31 | 转换脚本是否优先读到 `env_states/.../panda` |
| Smoke 好、仿真一直失败 | 除策略能力外，确认评测脚本使用 `robot.get_state()` 与训练一致 |
| `scene_json` 为空 | YOLO 阈值、画面内容；或先用 `--scene_source sim_gt` / `hybrid` 验证反思链路 |
| vLLM 连接失败 / 404 | `OPENAI_BASE_URL`、`--vlm_model` 与 `v1/models` 返回的 id 是否一致 |
| `git push` 403 | PAT 权限、`gh` 登录账号与仓库所有权；清掉错误的 `~/.git-credentials` 后重试 |
